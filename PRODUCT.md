# Product

## Register

brand

## Users

Developers who live in a terminal-based coding agent (Claude Code first, Cursor/Codex/Gemini CLI/Windsurf second). They find the site from GitHub, npm, or word of mouth, deciding in under a minute whether this tool is worth installing. Technical, skeptical of marketing, allergic to SaaS boilerplate. Secondary surface: the docs page, which the same people return to while integrating.

## Product Purpose

agent-sidecar gives a coding agent a visual canvas: the agent shows interactive HTML artifacts in the browser, and the user's clicks flow back into the conversation as input. The site exists to make that loop legible in seconds and get the two install commands copied. Success = visitor understands the loop without reading, installs, and can find any reference detail on the docs page.

## Brand Personality

Confident, technical, alive. An expert tool that enjoys itself — precise engineering voice, with the energy concentrated where the product concentrates it: light against dark. The emotional core is the moment a bright canvas opens beside a dark terminal.

## Anti-references

- The 2026 SaaS default: cream/sand body, terracotta accent, identical feature-card grids, eyebrow labels over every section. (The site's previous design — explicitly retired.)
- Generic "dev-tool dark mode": flat black, gray text, one neon gradient, mono everywhere as costume.
- Editorial-magazine affectation (display italic serifs, ruled columns) — wrong register for a tool.

## Design Principles

1. **The product is the picture.** The hero and diagrams show the actual loop (terminal ⇄ canvas ⇄ click ⇄ payload), not abstract decoration. Demo, don't describe.
2. **Light is the brand.** Dark ink world; artifacts and interactive elements are the light sources. Glow means "the canvas is on."
3. **Mono is literal, not costume.** Monospace appears only where a terminal or code genuinely appears; prose is set in the grotesque.
4. **Respect the reader's minute.** One idea per fold, copy-pasteable commands, no scroll-hijacking, docs one click away from everything.
5. **Same shell everywhere.** Landing and docs share one nav, one token set, one voice — no seams between marketing and reference.

## Accessibility & Inclusion

WCAG AA: ≥4.5:1 body text contrast (checked against the dark surfaces), ≥3:1 for large text; every animation has a `prefers-reduced-motion` alternative; keyboard-navigable nav and sidebar; semantic landmarks on both pages.
