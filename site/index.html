How it works

One loop, three beats

1

Claude shows

Claude calls create_artifact with any HTML — design options, a form, a diagram, a preview. It appears instantly on your canvas.

2

You click

Every artifact gets a claude.send() helper injected. Buttons, forms, anything interactive — your input becomes a payload.

3

Claude continues

Claude waits on await_interaction. Your click unblocks it mid-thought, and the conversation picks up right where you pointed.

Features

Small server, big surface

Live canvas

Artifacts stream to the browser over SSE. New ones take focus; updates hot-reload in place.

Webhook receiver

Anything that can POST — CI, scripts, curl — can drop events into your Claude session's queue.

Durable log

Every interaction lands in interactions.jsonl — replayable, tailable, greppable.

Plain MCP

No experimental channels, no org policy fights. Works anywhere Claude Code and an MCP server can run.

Localhost only

Binds 127.0.0.1. Your artifacts and your clicks never leave the machine.

One file

A single Bun process: MCP over stdio on one side, HTTP on the other. Read it in ten minutes.

Get started

Running in a minute

Install the plugin straight from GitHub — no clone, no build. Restart Claude Code and sidecar rides along in every project, opening the canvas the first time Claude has something to show you. Just have Bun on your PATH.

# in any Claude Code session
› /plugin marketplace add smarchetti/agent-sidecar
› /plugin install sidecar@agent-sidecar
# restart, then just ask
› show me three layout options…
